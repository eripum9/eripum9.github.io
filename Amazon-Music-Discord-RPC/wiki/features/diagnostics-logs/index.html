Diagnostics & Logs

The diagnostics window gives a quick overview of what is connected, what source is active, and what the app is doing.

What it shows

RPC status, Discord connection, metadata source, enhanced metadata state, log history, live console output, and issue reporting.

Development tests

The tests area is meant for development. The app warns normal users before showing it, and that warning can be hidden after acknowledgement.

Logs

Log rotation keeps older logs available while limiting growth. Older logs are removed after the configured history limit.

Report issue

The report issue button opens the GitHub issue page. Include version, Windows version, metadata source, Discord status, and redacted logs when useful.

Privacy notes

Diagnostics redacts known token values. Still review logs before posting screenshots or text publicly.
