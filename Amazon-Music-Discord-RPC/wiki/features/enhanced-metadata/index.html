Feature

Enhanced Metadata

Enhanced metadata is the recommended source because it reads richer playback details from the local Amazon Music desktop app.

Compatibility: Enhanced metadata is currently built and tested for the Microsoft Store version of Amazon Music. The website/desktop installer version may reject the metadata launch flag and show a --remote-debugging-port error. If that happens, install the Microsoft Store version or use fallback mode.

What it does

Improves title, artist, album, artwork, pause state, and timer accuracy by attaching to Amazon Music through a local metadata interface.

When to use it

Use it by default if you want the most reliable song identification and less dependence on Windows fallback metadata.

How to enable or disable it

Open Settings and toggle Enhanced Amazon metadata. New installs explain the feature before enabling it.

Repair behavior

If Amazon Music is already open without metadata access, the app can repair the session when automatic repair is enabled. This is meant to keep metadata reliable without manual steps.

Privacy and security notes

The debug port is selected randomly from a high local port range for each session, kept in memory, and the app only accepts Amazon Music targets on music.amazon.*.
